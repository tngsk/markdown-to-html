# Mono サーバー機能・コンポーネントマニュアル

Monoは、オフラインで動作する自己完結型HTMLを生成するツールですが、サーバー（`server.py`）を起動することで、リアルタイムでの状態同期やデータ収集などの高度な機能を利用できます。

本マニュアルでは、サーバーを利用する各コンポーネントの仕様と使用方法を解説します。

## 1. サーバーの概要と起動方法

Monoの同期・データ収集サーバーはFastAPIとWebSocketsを利用して構築されています。

### サーバーの起動
プロジェクトのルートディレクトリで以下のコマンドを実行します。

```bash
uv run server.py
```
デフォルトでは `http://0.0.0.0:8000` で起動し、以下の2つの主要なエンドポイントを提供します。

- **WebSocket (`/ws/sync`)**: `mono-sync` を通じて、ドキュメントのフォーカス（スクロール位置）をリアルタイムで同期します。
- **POST API (`/api/data`)**: 各コンポーネントからのイベントデータ（投票やリアクションなど）を受け取り、サーバー側の `data.jsonl` に保存します。

### 設定 (`config.toml`)
サーバーのセキュリティ設定（CORSや接続先）は `config.toml` の `[security]` セクションで管理されます。

```toml
[security]
connect-src = "http://localhost:8000"
ws-src = "ws://localhost:8000"
cors-allowed-origins = [
    "http://localhost:8000",
    "http://127.0.0.1:8000",
    "null" # ローカルファイル (file://) からのテストアクセス時に必要
]
max-upload-size = 1048576 # 1MB (デフォルトのDoS対策ペイロード上限)
```

## 2. サーバーと連携するコンポーネント一覧

### 2.1. `mono-sync` (同期・データ復旧ハブ)
`mono-sync` はサーバーと連携するための「ハブ」として機能する暗黙的コンポーネントです（通常、自動でページに組み込まれます）。

#### 機能
1. **フォーカス同期 (WebSocket)**
   - `?role=host` パラメータをつけて開いた画面（ホスト）のスクロール位置（見ている見出し：`h1, h2, h3`）を検知し、サーバーへ送信します。
   - `?role=participant` パラメータ等で開いた参加者の画面では、ホストが現在見ている箇所へ自動的にスクロールされます。
2. **データ収集の仲介**
   - ページ内の他のコンポーネント（例: `mono-poll`）が発行するイベント（`mono:vote` または `mono-data`）を検知し、API (`/api/data`) 経由でサーバーへ送信します。

---

### 2.2. `mono-export` (データエクスポート・同期)
ローカル（ブラウザの `localStorage`）に保存されたデータを一括でサーバーへ送信、またはJSONとしてダウンロードする機能を提供します。

#### 使用方法
CLIでの変換時に `--export` フラグを付与することで自動的に組み込まれます。
```bash
uv run main.py document.md --export
```

#### 機能
- 右下にフローティングボタンとして表示されます。
- **「Download JSON」**: オフライン環境でも、ブラウザに保存されたデータをJSON形式でダウンロードできます。
- **「Sync to Server」**: `localStorage` にあるコンポーネントのデータを一括で `/api/data` へ送信します。

---

### 2.3. インタラクティブなコンポーネント（データ送信元）

以下のコンポーネントは、ユーザーの操作に応じてイベント（`mono:vote` または `mono-data`）を発行し、それを `mono-sync` が拾ってサーバーへ送信します。通信ができないオフライン状態であっても、一旦ブラウザ（`localStorage`）に保存される設計になっています（Graceful Degradation）。

#### `mono-poll` (投票)
選択式の投票を行い、結果をサーバーへ送信します。
```markdown
@[poll](title: "好きな言語は？", options: "Python, JavaScript, Go")
```

#### `mono-reaction` (リアクション)
「いいね」などのリアクションボタンを表示し、クリック回数を記録・送信します。
```markdown
@[reaction](options: "👍, 👎")
```

#### `mono-notebook` (ノート・メモ入力)
ユーザーが自由記述できるテキストエリアです。入力内容は数秒のデバウンスを経て `localStorage` に保存されます。
```markdown
@[notebook](title: "メモ", placeholder: "自由に入力してください")
```

#### `mono-session-join` (セッション参加)
特定のセッションIDを入力し、ユーザーのセッションを確立します。
```markdown
@[session-join](title: "勉強会へ参加する")
```

---

## 3. 注意事項とトラブルシューティング

- **「オフラインモード」のログ**: サーバーが起動していない場合や、CORS設定で弾かれた場合、コンポーネントはブラウザのコンソールに「💡 [Mono] スタンドアロンモードで動作しています。データはローカルにのみ保存されます。」等のメッセージを出力し、エラーで画面が停止しないようにフォールバックします。
- **エクスポートの実行**: インタラクティブなコンポーネント（`mono-poll`など）がページに存在する場合、エクスポート機能は自動的に有効になる仕様です（`--export` なしでも出力されます）。
- **ペイロード上限**: `/api/data` への送信データは、デフォルトで 1MB (`max-upload-size`) に制限されています。これを超えるデータを送信しようとするとエラーになります。
