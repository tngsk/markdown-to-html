# Mono 拡張可能性リサーチ：外部ライブラリ・ツール・APIの活用シーン

本ドキュメントでは、Markdownから単一の自己完結型HTML（モノリシックドキュメント）を生成するMonoの特性と、独自の強力なディレクティブによるWeb Components展開機能を活かして、将来的に統合すると非常に興味深い・有用な外部ライブラリやツール、APIについて考察します。

Monoのコアバリューである「依存関係ゼロのポータビリティ」「オフライン動作の保証」を尊重しつつ、オプションとして組み込める（またはスクリプトをインライン化して同梱できる）技術を中心にリサーチしています。

---

## 1. インブラウザ演算・データ処理

Monoの「バックエンド不要で動く」という特性を極限まで高めるための技術です。

### 1.1. Pyodide / WebAssembly Python
- **概要**: WebAssemblyにコンパイルされたPythonランタイム。ブラウザ上で直接Pythonコードを実行できます。
- **Monoでの活用シーン**:
  - **インタラクティブなコード実行**: `@[python-runner]` のようなコンポーネントを作成し、ユーザーがブラウザ上でPythonコードを編集・実行・結果表示できるようにする。Jupyter Notebookのような体験を、Pythonバックエンドなしの完全な静的HTMLで提供可能になります。
  - **データ分析の配布**: データセット（CSV等をBase64埋め込み）と分析スクリプトを同梱したHTMLを配布し、受け取った側がローカルでパラメータを弄ってグラフを再描画するような用途。

### 1.2. SQL.js (SQLite compiled to WebAssembly)
- **概要**: CのSQLiteをWebAssemblyにコンパイルしたもので、ブラウザ内で完結するローカルデータベース。
- **Monoでの活用シーン**:
  - **インタラクティブなSQL学習教材**: `@[sql-console](db="sales.db")` のように記述。埋め込まれた初期データベース（Base64化）に対して、ユーザーがSELECT文などを発行し、結果をテーブル表示する。データベースの実態はメモリ上にあるため安全に体験可能。

## 2. インブラウザAI・機械学習

バックエンドAPIに依存しないローカルAIの活用です。

### 2.1. Transformers.js
- **概要**: Hugging Faceのモデルをブラウザ上で実行するためのJavaScriptライブラリ。
- **Monoでの活用シーン**:
  - **プライバシーセキュアなテキスト処理**: `@[ml-summarize]` や `@[ml-sentiment]` コンポーネント。Markdown内の長文をブラウザ上で要約・感情分析。データが外部サーバーに送信されないため、機密性の高い社内ドキュメントの配布と相性が良いです。
  - **オフラインでのセマンティック検索**: 配布された巨大なドキュメント内で、自然言語による意味検索（ベクトル検索）を完全オフラインで実現。

### 2.2. MediaPipe (Google)
- **概要**: 画像・動画から顔、手、姿勢などをリアルタイムに検出するオンデバイス機械学習ソリューション。
- **Monoでの活用シーン**:
  - **インタラクティブなプレゼン/教材**: Webカメラの映像を入力とし、手のジェスチャーでMono内のスライド（セクション）をめくる、または特定のポーズをとったことを判定する教育用コンテンツ。

## 3. 高度なビジュアライゼーション・ダイアグラム

Markdownによるテキスト記述からリッチな図を生成します。

### 3.1. Mermaid.js
- **概要**: テキストベースでフローチャートやシーケンス図、ガントチャートなどを描画するライブラリ。
- **Monoでの活用シーン**:
  - `@[mermaid]` コンポーネント。コードブロックの中にMermaid記法を書くことで、ビルド時またはブラウザでの表示時にSVGとしてレンダリング。Markdownの表現力を大幅に引き上げます。（※ビルド時にPythonライブラリを使ってSVG化し、HTMLに完全インライン化するアプローチがMonoの思想に最も適しています）

### 3.2. `<model-viewer>` (Google)
- **概要**: 3Dモデル（GLTF/GLB）を簡単にブラウザに埋め込み、AR表示もサポートするWeb Components。
- **Monoでの活用シーン**:
  - `@[3d-model](src="engine.glb")` のように記述。3Dモデル自体をBase64で埋め込むか、ローカルパスを指定。ハードウェアのマニュアルや、商品のプレゼン資料などを単一HTMLで配布し、受け取ったユーザーがぐりぐりと3Dモデルを回して確認できるようにする。

### 3.3. uPlot / Chart.js
- **概要**: 軽量で高速なグラフ描画ライブラリ。
- **Monoでの活用シーン**:
  - `@[chart](type="bar", data="data.csv")` のように記述し、Base64化されたCSVデータを読み込んで動的なグラフを描画。静的な画像と異なり、ホバーによるツールチップ表示やズームが可能になります。

## 4. 外部APIとの連携（ハイブリッドアプローチ）

Monoの基本は「オフライン・単一ファイル」ですが、オンライン環境を前提とすることで機能が拡張されるAPIの活用です。

### 4.1. Mapbox GL JS / Leaflet
- **概要**: インタラクティブな地図ライブラリ。
- **Monoでの活用シーン**:
  - `@[map](lat="35.68", lng="139.76", zoom="10")`。特定の場所を示すだけでなく、埋め込まれたGeoJSONデータ（店舗リストなど）を地図上にピン留めして表示。観光ガイドや不動産情報の配布用HTMLとして機能。

### 4.2. Web Speech API (ブラウザ標準)
- **概要**: 音声合成（Text-to-Speech）および音声認識。
- **Monoでの活用シーン**:
  - `@[read-aloud]` コンポーネント。外部ライブラリ不要で、Markdownのテキストをブラウザに読み上げさせるアクセシビリティ機能。
  - `@[speech-input]` コンポーネント。`mono-notebook` のような入力コンポーネントに対して、音声でテキストを入力させる。

## 5. 音楽・音響信号処理・コンポジション

Monoの「ローカルで完結する」特性は、遅延に敏感なオーディオ処理や、ブラウザのWeb Audio APIを活用した音楽生成・波形解析と非常に相性が良いです。

### 5.1. Tone.js
- **概要**: Web Audio APIの上に構築された、ブラウザ向けの強力な音楽生成フレームワーク。シンセサイザー、エフェクト、シーケンサーなどを提供します。
- **Monoでの活用シーン**:
  - `@[synth](type="fm", notes="C4,E4,G4", duration="4n")` のように記述し、ページ読み込み時やボタンクリックで指定の和音を鳴らすコンポーネント。
  - プログラマブルな音楽理論の教材として、テキストで記述したコード進行をその場でシンセサイザーとして再生させるインタラクティブな楽譜（`mono-score` との連携）。

### 5.2. MML (Music Macro Language) パーサー / Pico.js などの軽量音源
- **概要**: `C4D4E4` のようなテキストベースの文字列から音楽を生成する仕組み。
- **Monoでの活用シーン**:
  - `@[mml](tempo=120, data="T120 O4 L4 C D E F G A B <C")` のようなディレクティブ。Markdown内にMMLを直接書き込むことで、外部の音声ファイル（MP3等）をBase64で埋め込むよりも圧倒的にファイルサイズを節約しつつ、BGMやジングルを再生できます。「テキストから音楽が鳴る単一HTML」は非常にユニークな体験になります。

### 5.3. Wavesurfer.js
- **概要**: オーディオファイルの波形を描画し、再生・シーク・アノテーションを可能にするライブラリ。
- **Monoでの活用シーン**:
  - `@[waveform](src="interview.mp3", annotations="data.json")` のようなコンポーネント。Base64で埋め込まれた音声データの波形をCanvasに描画。特定の秒数にマーカーを引き、Markdown側のテキストと連動して再生箇所をジャンプするような「インタラクティブな音声文字起こし・インタビュー記事」の作成。

### 5.4. Meyda.js
- **概要**: ブラウザ上でオーディオ信号の特徴抽出（RMS, MFCC, Chromaなど）を行うライブラリ。
- **Monoでの活用シーン**:
  - マイク入力（または埋め込まれた音声）からリアルタイムに周波数スペクトルや波形データを抽出し、Canvasにビジュアライズする `@[audio-analyzer]` コンポーネント。音響工学や音声学のオフライン教材として、声の高さをその場で分析・表示するドキュメントを作ることができます。

## 6. 実装に向けたアーキテクチャの考察

これらの外部ライブラリをMonoに統合する際、「オフラインで動く単一ファイル」という原則をどう守るかが鍵となります。

1. **スクリプトのインライン化**:
   軽量なライブラリ（uPlotなど）は、変換時にソースコードを読み込み、最終的なHTMLの `<script>` タグ内に完全に展開してしまう（CDNを使わない）。
2. **事前レンダリング (Pre-rendering)**:
   Mermaid.jsのような図形生成は、可能であればMono（Python側）のビルドフェーズでSVGに変換してしまい、クライアントサイドのJS依存をなくす。
3. **グレースフル・デグラデーション**:
   CDN読み込みが必要なライブラリ（Pyodideなど容量が大きいもの）を使用する場合、オフライン時は「現在オフラインのためこのコンポーネント（Pythonエディタ等）は利用できません」というプレースホルダーを表示するフォールバックを実装する。

MonoのWeb Componentsベースのアーキテクチャ（`script.js`, `style.css`, `template.html` をカプセル化して注入する仕組み）は、これらの外部ライブラリを独立したコンポーネントとしてラップするのに非常に適しており、コアのコードベースを汚さずに強力な拡張を実現できます。
